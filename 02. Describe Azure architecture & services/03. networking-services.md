# AZ-900 Study Guide: Module 6 — Azure Networking Services

This module covers how resources talk to each other, to the internet, and to your on-premises environment. Networking questions on the AZ-900 exam love to test whether you know *which connection type fits which scenario* — so pay close attention to the comparison tables.

---

## 1. Azure Virtual Networking

**What it is (Simple Definition):**
Azure Virtual Networks (VNets) let Azure resources — like VMs, web apps, and databases — communicate with each other, with users on the internet, and with your on-premises computers. Think of it as an extension of your own network into Azure.

**The Everyday Analogy:**
Imagine your company has one office building (on-premises) and rents extra floors in another building across town (Azure). A virtual network is like the private hallway system connecting both buildings, so employees can move between them without ever stepping onto the public street.

**Why it matters (Key Benefit):**
Azure virtual networks provide:
- Isolation and segmentation
- Internet communication
- Communication between Azure resources
- Communication with on-premises resources
- Traffic routing
- Traffic filtering
- The ability to connect virtual networks to each other

**Public vs. private endpoints:**

| Endpoint Type | Description |
|---|---|
| **Public endpoint** | Has a public IP address; accessible from anywhere in the world |
| **Private endpoint** | Exists within a virtual network with a private IP address from that network's address space |

### Isolation and Segmentation

You define a private IP address space for your virtual network, then divide it into subnets, allocating part of that address space to each one. This IP range only exists within the virtual network and isn't internet-routable. Name resolution can use Azure's built-in service or a custom internal/external DNS server.

### Internet Communication

You can allow incoming internet connections by assigning a public IP address to a resource, or by placing it behind a public load balancer.

### Communicating Between Azure Resources

Two main approaches:
- **Virtual networks** connect VMs and other resources like App Service Environment, Azure Kubernetes Service, and VM scale sets.
- **Service endpoints** connect to resource types like Azure SQL databases and storage accounts, improving security and routing between resources.

### Communicating with On-Premises Resources

| Connection Type | How It Works |
|---|---|
| **Point-to-site VPN** | An individual computer initiates an encrypted VPN connection into the Azure virtual network |
| **Site-to-site VPN** | Links your on-premises VPN device/gateway to an Azure VPN gateway, making Azure resources appear as part of the local network |
| **Azure ExpressRoute** | A dedicated private connection to Azure that never touches the public internet — useful for higher bandwidth and security needs |

### Routing Network Traffic

By default, Azure routes traffic automatically between subnets, connected networks, and the internet. You can override this with:
- **Route tables:** Custom rules for how packets move between subnets.
- **Border Gateway Protocol (BGP):** Works with VPN gateways, Route Server, or ExpressRoute to propagate on-premises routes into Azure.
- **User-defined routes (UDR):** Give you granular control over traffic flow between subnets or virtual networks.

### Filtering Network Traffic

- **Network security groups (NSGs):** Contain inbound/outbound rules that allow or block traffic based on source/destination IP, port, and protocol.
- **Network virtual appliances:** Specialized VMs acting like a hardened network appliance — for example, running a firewall or WAN optimization.

### Connecting Virtual Networks

**Virtual network peering** connects two virtual networks directly. Traffic between peered networks stays private on the Microsoft backbone network and never touches the public internet. Peered networks can even span different regions, letting you build a global interconnected network.

**Exam Takeaway:** Virtual networks handle isolation, routing, and filtering — peering lets separate virtual networks (even across regions) talk to each other privately.

## 2. Azure Virtual Private Networks (VPN)

**What it is (Simple Definition):**
A VPN creates an encrypted tunnel across an untrusted network (usually the public internet), connecting two or more trusted private networks safely.

**The Everyday Analogy:**
Think of a VPN like an armored, unmarked van driving through public streets. The streets (internet) are open to everyone, but what's inside the van (your data) is protected the entire way.

### VPN Gateways

A VPN gateway is a type of virtual network gateway, deployed in a dedicated subnet, that enables:
- **Site-to-site** connections (on-premises datacenter to virtual network)
- **Point-to-site** connections (individual device to virtual network)
- **Network-to-network** connections (virtual network to virtual network)

**Key facts:**
- You can deploy only one VPN gateway per virtual network, but one gateway can connect to multiple locations.
- Authentication uses a preshared key regardless of VPN type.

**Policy-based vs. route-based gateways:**

| Type | How It Decides What to Encrypt |
|---|---|
| **Policy-based** | Statically matches packet IP addresses against defined sets to choose the tunnel |
| **Route-based** | Models IPSec tunnels as network interfaces; IP routing decides which tunnel to use |

Route-based VPNs are preferred for on-premises connections and are more resilient to topology changes. Use a route-based gateway if you need:
- Connections between virtual networks
- Point-to-site connections
- Multisite connections
- Coexistence with an ExpressRoute gateway
- High-availability scenarios

### High-Availability Options for VPN Gateways

| Option | How It Works |
|---|---|
| **Active/standby** (default) | Two instances behind the scenes; standby takes over automatically during failover — typically a few seconds for planned maintenance, up to 90 seconds for unplanned disruptions |
| **Active/active** | Each instance gets its own public IP with separate tunnels, using BGP routing for added resilience |
| **ExpressRoute failover** | A VPN gateway acts as a backup path if an ExpressRoute circuit has an outage |
| **Zone-redundant gateways** | Deployed across availability zones for resilience against zone-level failures (requires specific gateway SKUs and Standard public IPs) |

**Exam Takeaway:** VPN gateways are deployed active/standby by default — you don't need extra configuration to get basic failover protection.

## 3. Azure ExpressRoute

**What it is (Simple Definition):**
ExpressRoute extends your on-premises network into the Microsoft cloud through a private connection (called an ExpressRoute Circuit), set up with the help of a connectivity provider — completely bypassing the public internet.

**The Everyday Analogy:**
If a VPN is an armored van driving on public roads, ExpressRoute is a private, dedicated highway built just for you — no traffic lights, no other drivers, and no exposure to the public roads at all.

**Why it matters (Key Benefit):**
- More reliable, faster, and more secure than typical internet connections, since it never touches the public internet.
- Connects to Microsoft cloud services like Microsoft 365, Dynamics 365, Azure Virtual Machines, Cosmos DB, and Azure Storage — across all regions.
- **ExpressRoute Global Reach** lets you connect multiple ExpressRoute circuits together, so, for example, an office in Asia and a datacenter in Europe can communicate without using the public internet.
- Uses **BGP** for dynamic routing between your network and Microsoft's.
- **Built-in redundancy** at every peering location, with the option to configure multiple circuits for extra resilience.

**When to choose ExpressRoute:**
- You need private, consistent connectivity between on-premises networks and Azure.
- Your team has strict compliance or data-transfer requirements.
- You need predictable latency and high throughput.
- You want to avoid sending critical traffic over the public internet.

> **Important security note:** Even with ExpressRoute, some traffic — like DNS queries, certificate revocation checks, and Azure CDN requests — still travels over the public internet.

**Exam Takeaway:** ExpressRoute is the private, dedicated connection option — choose it over a VPN when you need guaranteed bandwidth, consistent latency, and to avoid the public internet entirely.

## 4. Azure DNS

**What it is (Simple Definition):**
Azure DNS is a hosting service for DNS domains, providing name resolution using Microsoft's global infrastructure — managed with the same credentials, tools, and billing as your other Azure services.

**The Everyday Analogy:**
Think of DNS like a phone book that translates a name (like a website address) into a number (an IP address) so devices know where to connect. Azure DNS is Microsoft's globally distributed version of that phone book, built for speed and reliability.

**Why it matters (Key Benefit):**
- **Reliability and performance:** Hosted on Azure's global network of DNS name servers, using anycast networking so the closest available server answers each query.
- **Security:** Built on Azure Resource Manager, giving you Azure RBAC, activity logs, and resource locks.
- **Ease of use:** Manage records through the Azure portal, PowerShell, Azure CLI, or integrate via REST API and SDKs — using the same credentials and billing as your other Azure services.
- **Customizable virtual networks:** Supports private DNS domains, letting you use your own custom domain names within private virtual networks.
- **Alias records:** Point to an Azure resource (like a public IP, Traffic Manager profile, or CDN endpoint) and automatically update if that resource's underlying IP address changes.

> **Important:** Azure DNS doesn't sell domain names. You still need to purchase a domain through App Service domains or a third-party registrar — Azure DNS is only for hosting and managing records once you own the domain.

**Exam Takeaway:** Azure DNS handles record management and resolution, not domain name purchases — remember that distinction for exam questions.

## Quick Module Recap

- **Virtual networks** provide isolation, routing, and filtering; **peering** connects virtual networks privately, even across regions.
- **VPNs** create encrypted tunnels over the public internet — point-to-site, site-to-site, and route-based vs. policy-based are the key distinctions.
- **ExpressRoute** provides a private, dedicated connection to Azure that bypasses the public internet entirely, ideal for compliance, bandwidth, and latency-sensitive needs.
- **Azure DNS** hosts and manages DNS domains using Azure's global infrastructure, but doesn't sell domain names itself.
