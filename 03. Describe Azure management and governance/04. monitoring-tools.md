# AZ-900 Study Guide: Module 12 — Azure Monitoring Tools

This module covers the tools Azure gives you to stay informed — whether that's getting proactive recommendations, checking on Azure's own health, or digging into detailed logs and metrics from your own resources.

---

## 1. Azure Advisor

**What it is (Simple Definition):**
Azure Advisor evaluates your Azure resources and gives you personalized recommendations to improve reliability, security, performance, and cost efficiency — built right into the Azure portal.

**The Everyday Analogy:**
Think of it like a personal trainer who reviews your workout routine and gives specific, actionable feedback: "add more reliability here," "cut this cost there," "tighten up this security gap." You can act on the advice right away, save it for later, or dismiss it if it doesn't apply.

**Why it matters (Key Benefit):**
- Each recommendation includes a suggested action you can take immediately, postpone, or dismiss.
- You can set up notifications so Advisor alerts you as new recommendations appear.
- The dashboard covers all your subscriptions, filterable by subscription, resource group, or service.

**The five recommendation categories:**

| Category | What It Covers |
|---|---|
| **Reliability** | Flags configuration risks that could affect application uptime |
| **Security** | Detects threats and vulnerabilities that could lead to breaches |
| **Performance** | Identifies changes that could speed up your applications |
| **Operational Excellence** | Suggests workflow and deployment improvements |
| **Cost** | Finds ways to reduce your Azure spending |

**Exam Takeaway:** Azure Advisor is proactive and personalized — it tells you what to fix before it becomes a problem, across five specific categories.

## 2. Azure Service Health

**What it is (Simple Definition):**
Azure Service Health keeps you informed about the health of Azure itself and the specific resources you run, using three views that narrow in scope from global down to a single resource.

**The Everyday Analogy:**
Think of it like checking the weather at three different zoom levels: a national weather map (Azure Status), your local city forecast (Service Health), and a sensor on your own back porch (Resource Health) — each one gives you more specific, relevant detail than the last.

**The three health views:**

| View | Scope | What It Tells You |
|---|---|---|
| **Azure Status** | Global | Overall health of Azure across all services and regions — check here if you hear about a widespread outage |
| **Service Health** | Your services and regions | Outages, planned maintenance, and health advisories relevant to what you actually use; supports automatic alerts |
| **Resource Health** | Individual resource | Whether a specific resource (like a VM) is running normally, and whether an issue is on Azure's side or yours |

**Why it matters (Key Benefit):**
- Used together, these views give visibility from broad service-wide events down to a single resource.
- Historical alerts are retained, so you can spot recurring trends.
- When an event affects your workload, Service Health provides direct links to support so you can respond quickly.

**Exam Takeaway:** Azure Status is Azure-wide, Service Health is personalized to your environment, and Resource Health zooms into a single resource — remember the order of scope from broad to narrow.

## 3. Azure Monitor

**What it is (Simple Definition):**
Azure Monitor is a platform for collecting, analyzing, and acting on data from your Azure resources and applications, working across Azure, on-premises, and multicloud environments.

**The Everyday Analogy:**
Think of Azure Monitor like a building's central security and diagnostics system — cameras and sensors (logs and metrics) feed data from every floor into one control room, where staff can review footage, spot patterns, and set off alarms when something needs attention.

**Why it matters (Key Benefit):**
- Gathers logs and metrics from applications, operating systems, and network layers.
- Stores that data centrally, then makes it available through dashboards, queries, and alerts.

### Azure Log Analytics

**What it is:** The tool within the Azure portal where you write and run queries against data collected by Azure Monitor.

**Why it matters:** You can do simple filtering (e.g., find all errors in the last hour) or run advanced analytics to visualize trends over time.

### Azure Monitor Alerts

**What it is:** Notifications triggered when Azure Monitor detects that a defined condition has been met.

**How it works:**
- You create an **alert rule** specifying the condition.
- You create an **action group** controlling who gets notified and what happens next.

**Two types of alerts:**

| Type | Example |
|---|---|
| **Metric-based** | Send an email when a VM's CPU stays above 80% |
| **Log-based** | Watch for a specific error pattern across multiple resources |

> **Good to know:** Action groups are reusable across Azure Monitor, Service Health, and Azure Advisor — you don't need to recreate notification logic for each tool.

### Application Insights

**What it is:** An Azure Monitor feature that monitors the performance and usage of your web applications, whether they run in Azure, on-premises, or another cloud.

**Why it matters (Key Benefit):**
- Set up by adding an SDK to your code, or by enabling the Application Insights agent without any code changes.
- Monitors request rates, response times, and failure rates.
- Tracks dependency calls and their performance.
- Measures page load times, user counts, and session trends.
- Reports server performance counters like CPU, memory, and network usage.
- Supports availability tests — synthetic requests sent to your application to confirm it's responding, even during low-traffic periods.

**Exam Takeaway:** Azure Monitor is the overall data platform; Log Analytics is where you query that data; alerts notify you when conditions are met; Application Insights specifically focuses on application performance and usage.

---

## Quick Module Recap

- **Azure Advisor** gives personalized, proactive recommendations across five categories: reliability, security, performance, operational excellence, and cost.
- **Azure Service Health** narrows from global (Azure Status) to your environment (Service Health) to a single resource (Resource Health).
- **Azure Monitor** collects logs and metrics across Azure, on-premises, and multicloud environments.
- **Log Analytics** lets you query that collected data; **alerts** (metric- or log-based) notify you when defined conditions are met; **Application Insights** focuses specifically on web application performance and usage.
